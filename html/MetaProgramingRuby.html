Programming can seem like magic, with programs seeming to be able to do so many things. I personally believe that the art of programming is bassically the process of automating away pretty much everything. Wherever there is data, a programming language can be utilised to munipulate that data and cause magic to happen.

A program is just intructions passed into a compiler. So, could you create a program to munipulate a program? Yes, you can! This is bassically what we mean by Metaprogramming, when you have a program that munipulates the set of instructions. A program that programs a program, very meta indeed.

Metaprogramming is writing code that writes code.

So, now we know what metaprogramming is. It might sound like an advanced topic, only relevant in a small set of circumstances. But this isn't true. Case in point: compilers. Yes compilers take human readable instructions and convert them into binary code, so given a set of instructions a compiler will write binary code. So you cant say machines can't program, even a program like a compiler can program. In fact any code generator or compilation depends on metaprogramming, so metaprogramming is in fact everywhere.

colors =
[data*].map do |color| "\n.#{color.downcase}, .#{color} {" +
    "\n\tcolor: #{color};\n}\n"
end

# create file color.css if not there
# w+ means write and read access to file 
File.open("color_new.css", "w+"){
  |file|  file.puts "#{colors.reduce(:+)}"
}

*data copied from W3 schools

Now run the file with ruby

  $ ruby metaprogramming.rb

Congratulation you just ran a program to create this css file. Using ruby to create a file in this case the CSS file is all that Metaprogramming entails.

Conclusion

This brings us right back to the title of this article. Because Ruby is probably the most metaprogramming of all common languages, there is no barriers in Ruby to take advantage of metaprogramming.

Just think about how Ruby on Rails works, you use the generator to generate entire project scafffolds and activeRecord is bassically using Ruby to generate a lot of SQL code and queries.

In fact consider this, in Ruby metaprogramming is so deeply part of the language itself that you find thats its not distinguishable from normal programming, to be a Ruby programmer is to do metaprogramming consistently and routinly.
